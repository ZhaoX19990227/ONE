# ONE 0.5 产品范围

## 定位

ONE 不是外卖平台、点评平台或减肥工具，而是“吃喝决策 + 生活记忆”小程序。首发仅服务一个微信群，不做公开广场、陌生人匹配和多群运营。

## 三个维度

- `MEAL`：吃饭推荐、转盘、记录、评价、金额、统计与分享。
- `MILK_TEA` / `COFFEE`：饮品推荐、品牌与产品记录、糖冰小料、金额、统计与分享。
- `PRIVATE_HABIT`：界面名“鹿一下”，只记录时间、次数和可选身体感受，不接入 AI，不进入分享。

## 导航

1. **转一转**：今日时间线、吃饭转盘、奶茶咖啡转盘、鹿一下。
2. **日历**：按天查看缩略图/品牌标识，按维度查看日周月统计，生成海报。
3. **我的**：头像昵称、可选身高体重与属性、口味、预算、忌口、隐私和 AI 设置。

“我的”内提供独立的记忆中心：用户能看到 ONE 为什么记住某个偏好，并可以主动忘记任意一条。记忆不是不可解释的画像标签。

## 推荐记忆

推荐不是无状态随机。每次反馈会形成一条可追溯记忆：

```text
2026-07-17 · 喜茶 · 多肉葡萄 · 五分糖
反馈：有点甜
记忆：同产品下次优先建议三分糖；同类果茶降低默认甜度。
```

再次推荐时展示：

> 建议少少甜。7月17日你觉得多肉葡萄五分糖有点甜。

记忆类型包括喜欢、不喜欢、有点甜、太甜、有点咸、太辣、分量偏小、分量偏大、价格偏贵、值得回购和暂时屏蔽。

## 两条录入路径，一个事实源

### 推荐路径

`生成候选 → 用户选择 → 预填记录 → 确认实际吃喝/金额/配置 → 保存`。用户只选择“就它了”但没有确认实际记录时，保留为决策历史，不进入日历与消费统计。

### 自主记录路径

`拍照上传 → AI识别品牌/品类/产品 → 用户确认或修正 → 保存`。识别失败时依次引导选择已有品牌、品类和产品；仍未命中时允许自定义名称，并进入待归一化目录。

两条路径最终都写入同一个 `life_record` 事实源，并共同参与日历、金额统计、偏好记忆和后续推荐。记录保留 `RECOMMENDATION/MANUAL/AI_PHOTO` 来源，用于推荐采纳率分析，不影响用户看到的时间线。

已保存记录支持“再记一次”和删除。删除会同步停用由该记录产生的口味记忆，避免误记持续污染推荐。

## 动态主题

- 清晨：奶油杏与浅珊瑚。
- 午间：暖白、暧昧粉与雾霾蓝。
- 傍晚：落日粉与灰蓝。
- 深夜：深梅紫、低饱和粉蓝与半透明深色卡片。

切换使用全局设计令牌和柔和渐变，不闪屏，不改变信息层级。

## 成功指标

- 推荐采纳率：实际记录与推荐候选一致的比例。
- 记忆命中率：带记忆提示的推荐被采纳比例。
- 决定后记录完成率。
- 周/月复用率。
- 吃喝海报分享打开率。
