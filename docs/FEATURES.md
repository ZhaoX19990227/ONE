# ONE 功能现状

更新时间：2026-07-20。本文是产品、前端、后端共同使用的功能基线。

## 已完成的用户闭环

### 1. 转一转

- 吃饭、奶茶、咖啡三个推荐维度，支持转盘和记忆推荐。
- 推荐结合当前时段、目录基础权重、个人口味标签、历史反馈和可选单次预算。
- 候选展示品牌、产品、参考价和可解释原因；选中后进入预填记录页。
- 转盘由后端预先确定真实结果，前端按扇区中心以先快后慢曲线精准落点，动画结束后才确认选择。
- “不想吃/最近吃过/这次有点贵”形成 14 天弱负反馈；“换一批”生成新候选，两者都不创建生活记录。
- “鹿一下”只做轻量计数和动态回应，不进入 AI、消费统计或分享海报。
- 首页展示今日记录、缩略图或品牌色标、金额、备注；点击记录可再次记录或删除。
- 关闭“首页显示鹿一下”后，首页入口和今日私密卡片同时收起，历史数据不丢失。

### 2. 记录与图片识别

- 推荐确认、自主手填、拍照识别最终写入统一 `life_record` 数据源。
- 图片上传后由 Qwen3.6 多模态识别品牌、品类和产品，给出候选与置信度。
- 识别失败或目录未命中时允许用户修正/自定义，避免流程卡死。
- 支持实际金额、图片、备注、评分、是否回购和口味反馈。
- 奶茶/咖啡支持甜度与冰量/温度记录。
- 保存后即时生成动态回应；防重复提交并校验异常金额。

### 3. 可解释记忆

- “有点甜/太甜”形成降甜建议；“还会再来/下次不了”调整后续推荐权重。
- 推荐卡片会展示记忆来源，而不是只给一个黑盒结果。
- 记忆中心支持按吃饭、奶茶、咖啡筛选，展示来源日期和下次建议。
- 用户可单独忘记一条记忆；删除来源记录时会自动停用关联记忆。
- 关闭“让推荐参考口味记忆”后，转盘仍可用，但后端不读取历史记忆。

### 4. 日历、统计与分享

- 月历按吃饭、奶茶、咖啡、鹿一下显示不同圆点，并可进入某日时间线。
- 展示月活跃天数、吃喝金额、分类次数与金额、常去品牌排行和自然语言观察。
- 每月预算显示使用进度与剩余额度/超出提示，只提醒、不评判。
- 月报海报不包含“鹿一下”；生成后可预览、保存相册或发给群友。
- 日历内记录同样支持“再记一次”和删除，删除后月历与统计立即刷新。

### 5. 我的

- 微信头像昵称、身高、体重、1/0/0.5/side、吃饭偏好、饮品偏好和月预算。
- 口味标签会参与后端推荐打分，设置不再只是展示字段。
- 可控制口味记忆是否参与推荐，以及首页是否显示“鹿一下”。
- 资料、口味记忆和私密记录默认仅自己可见。

### 6. 体验与视觉

- 清晨、午间、傍晚、深夜四套动态主题。
- 暖白、暧昧粉、雾霾蓝的低饱和色彩；毛玻璃卡片、柔和阴影和有机背景光斑。
- 页面进场、底部弹层、按钮按压、卡片触感、轻震动、转盘和花瓣反馈。
- 加载、空态、识别失败、保存失败和相册权限均有承接反馈。

### 7. 群友今晚吃什么

- 从推荐候选一键创建 24 小时房间，通过微信分享卡片邀请群友。
- 支持投票、改票、实时票数进度与发起人揭晓；并发改票通过数据库行锁和唯一约束兜底。
- 房间只返回候选、聚合票数、自己的票和赢家，不公开谁投了谁，也不会读取个人吃喝记录。

### 8. 温和周报

- 展示本周生活切片、吃喝金额和自然连续天数，可生成周报海报。
- 7/14 天等节点只出现一次轻彩蛋，文案强调连续不是任务，不提供补签、红点或断签惩罚。

## 已完成的工程能力

- Java 21、Spring Boot 4.1、MySQL 8.4、Flyway 单一版本初始化。
- 微信 code 登录、Bearer Token、用户数据按 `userId` 隔离。
- 记录软删除，保留审计数据；业务查询只读取 `CONFIRMED`。
- Qwen3.6 模型、API 地址和密钥全部环境变量化；无密钥时安全降级到手动确认。
- DeepSeek 仅增强记忆推荐的动态回应与三候选轻量重排；严格校验候选 ID、限制文案长度、短超时并自动回退本地推荐，普通转盘不依赖 AI。
- Docker Compose + Caddy HTTPS，适合一台阿里云 ECS 的低成本首发。
- 自定义品牌/产品归一化管理 API：批量合并同名待处理项、自动沉淀品牌别名，后续录入会直接命中标准目录。
- 本地磁盘与阿里云 OSS 可配置切换；OSS 自动生成 320px WebP 处理地址，上传链路可接阿里云内容安全。
- Actuator + Prometheus 指标、容器健康检查与 5xx/JVM/数据库连接池告警规则；公网 Caddy 明确屏蔽指标端点。

## 上线前仍需配置

- 微信小程序 AppID/AppSecret、合法 request/uploadFile 域名与隐私协议。
- DashScope 与 DeepSeek API Key；建议先使用低成本模型，通过环境变量切换模型版本。
- 公网 HTTPS 域名、Caddy 邮箱、MySQL 强密码、Token Secret。
- 生产建议设置 `ONE_STORAGE_TYPE=oss` 并启用内容安全；RAM 权限只授予目标桶，不使用主账号 AccessKey。
- 小程序审核文案与类目需要进一步确认，尤其是“鹿一下”的最终呈现。
